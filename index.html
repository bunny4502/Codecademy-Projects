1. Basic Elements

View the table I have created below to see the elements names, how they appear in code, and what they do/mean!

Element/Attribute Code Purpose/Meaning
Form <form> This element signifies the beginning of a form and contains all of its child elements until it is closed with </form>
Action action="./link" This attribute is set to the value of the link (either external or relative path) where submissions should go.
Method method="POST" This attribute is set to the value of the https method we want to use. This will usually be POST.

2. Types of Inputs and How to Use Them

So much different information can be collected with a form, and to make it as user-friendly as possible we want to supply our users with the most logical "input" field to enter information into. Take a look at the table below to see the input types, how they appear in code, and what they do/mean.

Element Code Purpose/Meaning
Input <input> Renders a box for user input. Needs to have For= attribute assigned to it with a value that matches the Label to associate them.
Label <label> Label renders the text to accompany the input box. Must share the same for="value" as input element. Text is written directly between opening and closing tags.
Text <input type="text"> Renders a text field that can be typed into.
Number <input type="number"> Renders a text box that will ONLY accept numbers.
Password <input type="password"> Automatically masks inputs as dots or astericks.
Submit <input type="submit" value="Submit"> Renders the submit button, text will render as whatever text the value is set to.

3. Form Validations

Speaking of user-friendly, we want to make sure our users don't make any obvious mistakes when they input information into our forms! Let's look at validation for that (think of it as adding parameters to your input fields)!

Let's look at different parameters we can set:


  • Make a Field Required: add a required attribute to an input element.
  • Set a Minimum/Maximum: when an input element has a type="number" attribute, we can add additional min="#" and/or max="#" attributes.
  • Set Text Length Requirements: we can add additonal minlength="#" and/or maxlength="#" attributes to text inputs that set minimum/maximum # of characters that can be entered.
